# Anti-Patterns — What Kills Names

The fastest way to improve naming output is to learn what bad looks like. These patterns are so common they've become invisible to the people producing them — which is exactly why they fail.

---

## AI Name Slop

AI-assisted naming produces polished-but-interchangeable results. A "Sea of Sameness." These patterns are red flags that a name was generated by a language model rather than crafted through genuine metaphor exploration:

| Pattern | Examples | Why it's dead |
| --------- | ---------- | --------------- |
| **Suffix addiction** | Glidely, Parsify, Chatible, Syncora, Buildly | Every AI uses the same -ly, -ify, -able, -ora, -ix suffixes. Thousands of startups already did this. |
| **Meaningless portmanteau** | Brightrix, Nexagen, Vaultora, Intellicore | Technically unique but indistinguishable from each other. No story, no image. |
| **Thesaurus extraction** | Apex, Pinnacle, Summit, Zenith, Catalyst | Millions of companies already used these. The thesaurus is not a naming tool. |
| **Category + modifier** | CloudGuard, DataSync, SiteWatch, CodeForge | Descriptive but forgettable. Tells you what it does, gives you nothing to remember. |
| **Vacant corporate language** | [Anything] Solutions, [Anything] Labs, [Anything] Platform, [Anything] HQ | Empty calories. Communicates nothing except "we are a company." |
| **Excessive misspelling** | Replacing every vowel, random K-for-C swaps | Worked briefly around 2010. Now signals "the real spelling was taken." |
| **Fake Latin/Greek mashups** | Cognivex, Synaptiq, Quantera, Luminex | Sounds impressive for 2 seconds, then you realize it means nothing. |
| **Aspirational adjectives** | BrightPath, TrueNorth, ClearView, SwiftBase | Generic positivity. Could be any product in any category. |

**The core problem:** AI remixes what already exists in its training data. It can't produce genuinely novel naming because novel names by definition don't appear in training data. Use AI for brainstorming volume (generate 200 candidates), never for final selection. The final name should come from human metaphor exploration and judgment.

**How to spot slop:** If you can swap the name between three different products in different categories and it still "works," it's slop. A good name is specific to its product.

## The Five Fatal Flaws

Framework adapted from professional naming agencies. If a name has any of these, it's dead:

### 1. Boring Descriptive Names

Driven by fear of standing out. "WordPress Site Monitor" is a description, not a name. "Cloud-Based Team Collaboration Platform" is a sentence, not a brand.

**Why it happens:** Stakeholders feel safe with descriptions because they're immediately clear. But clarity without memorability is useless — people understand what the product does for 5 seconds, then forget its name forever.

**The fix:** Use the description as your tagline, not your name. The name creates recall; the tagline creates clarity.

### 2. Invented Mashups With No Meaning

Technically unique but functionally identical to each other. Syncora, Brightrix, Nexagen, Cognivex — try telling these apart from memory after seeing them once.

**Why it happens:** Founders want an available .com domain. Invented words are available because nobody wants them. That's not a feature.

**The fix:** Start from a real word or recognizable morpheme and modify it, rather than assembling random syllables.

### 3. Thesaurus-Derived Names

Interchangeable with every competitor who opened the same thesaurus page. Apex, Pinnacle, Summit, Zenith — these words are so overused as brand names that they've lost all meaning.

**Why it happens:** The brainstorming process starts with "what words mean [our category]?" instead of "what metaphors connect to what we do?"

**The fix:** Never open a thesaurus looking for synonyms of your product's main attribute. Instead, explore metaphor territories — what else in the world works like your product?

### 4. Empty Corporate Language

Solutions, Network, Systems, Platform, Labs, Group, Global, Enterprise, Digital, Technologies — words that pad a name without adding information.

**Why it happens:** It "sounds professional." But sounding professional and being memorable are different goals, and the corporate language words achieve neither.

**The fix:** If removing a word from the name doesn't change its meaning, that word is filler. Remove it.

### 5. Extremeness For Its Own Sake

Trying too hard to be edgy, bold, or unique without strategic purpose. DEATHKRYPT ULTRA for a to-do app. THUNDERFORGE for a calendar tool.

**Why it happens:** Overcompensating for the fear that a name is "too boring." But intensity without relevance is just noise.

**The fix:** The name's intensity should match the product's character. A monitoring tool can be intense. A note-taking app probably shouldn't be.

## Common Compound Name Traps

### Filler-Word First Position

These words in the first position of a compound name are almost always filler:

Smart, Auto, Cloud, AI, Data, Cyber, Next, Digital, True, Pure, Open, Fast, Easy, Pro, Ultra, Hyper, Super, Meta, Omni, Core

They've been prepended to thousands of products. They communicate nothing specific. If your compound starts with one of these, the second word is doing all the work — so just use the second word.

### Generic Second Position

These words in the second position are similarly empty:

Hub, Lab(s), Base, Flow, Sync, Link, Stack, Forge, Shift, Scape, Verse, Craft, Wise, Path, Nest, Box

"SmartHub" = filler + filler. Neither word carries meaning. The compound is worse than either word alone.

### The Compound Test

For any compound name candidate:
1. Cover the first word. Does the second word tell you something specific?
2. Cover the second word. Does the first word tell you something specific?
3. Can you replace either word with a synonym and get an equally "good" name?

If #3 is yes, the name has no specificity. Kill it.

## Red Flags Checklist

Ask these about any name candidate. If the answer is yes, kill the name:

- [ ] Does someone hear it and say "oh, like [10 other products]"?
- [ ] Can you NOT tell a 15-second story about why it's called that?
- [ ] Do people need to spell it out letter by letter after hearing it?
- [ ] Does it only work visually but is awkward spoken aloud?
- [ ] Is it just a "cool word" with no metaphor connecting to the product?
- [ ] Would swapping it to a different product in a different category still "work"?
- [ ] If you remove any stylization (ALL CAPS, special characters, letter substitution), does the name become completely generic?
- [ ] Does it match three or more patterns in the AI Name Slop table above?
- [ ] Would you cringe introducing the product by name in a professional context?
- [ ] Is the name's primary virtue that the domain was available?
- [ ] Does any common meaning or connotation of this word **conflict with** the product's core function or value proposition?

Two or more "yes" answers = disqualify the candidate.

## Semantic Self-Contradiction

A name can have a strong primary metaphor that maps well to the product, while simultaneously carrying a secondary meaning that undermines the product's core story. This is a subtle failure mode — the name has a story, plants an image, sounds good, and is distinctive, but it carries a built-in contradiction.

Check all dictionary definitions and common usages of the candidate word — not just the one you're using as the metaphor. If any common meaning contradicts the product's purpose, the name will confuse people who encounter the contradictory meaning first.

**How to catch it:** For each candidate, ask: "What else does this word mean?" Look at all common definitions, idioms, and phrases that use the word. If any of them work against the product's core value, the name is weakened — even if the primary metaphor is strong.

---

## The "So What?" Test

After explaining a name's origin to someone, their response tells you everything:

- **"That's clever"** → The name works. The metaphor landed.
- **"Oh, I see"** → Acceptable. The connection exists but isn't exciting.
- **"So what?"** → The name fails. The story doesn't earn its keep.
- **"I don't get it"** → The name fails. The reference is too obscure.
- **"Isn't that already a thing?"** → The name fails. It's occupied territory.

Aim for "that's clever." Settle for "oh, I see." Never ship "so what?"
